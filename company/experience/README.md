# company/experience/ — Past project summaries

Create one markdown (`.md`) file per completed project.
Claude selects the most relevant projects when drafting proposals.

## File naming

Use descriptive names: `clientname-projecttype-year.md`
Example: `acme-bank-payment-gateway-2024.md`

## What to include in each file

Copy and fill this template:

```
# Project: [Short project name]

## Client
Name: [Client organisation name]
Sector: [e.g. Banking, Government, Healthcare]
Contact (if referenceable): [Name, designation, email/phone]

## Project overview
[2–4 sentences describing what was built or delivered]

## Scope of work
- [Bullet 1]
- [Bullet 2]

## Contract value
[Approximate value — e.g. ₹2.5 Cr or $300,000]

## Duration
[Start month/year] to [End month/year] ([X months])

## Team size
[Number of people deployed on this project]

## Technologies / platforms used
[List key technologies]

## Key outcomes / measurable results
- [e.g. Reduced transaction processing time by 40%]
- [e.g. Onboarded 50,000 users in first month]

## Reference letter / completion certificate available?
[Yes / No / In progress]
```

## Also upload to `documents/`
If you have a completion certificate or client reference letter as a PDF,
upload it to `company/documents/` and mention the filename in the project file above.
