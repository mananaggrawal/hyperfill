# company/about/ — Company capability descriptions

Create plain text (`.md`) files here describing your company.
Claude reads these when writing proposals and evaluating fit for an RFP.

## Suggested files to create

| File | What to write |
|---|---|
| `overview.md` | What your company does, your mission, when founded, key markets served |
| `technical-capabilities.md` | Technology stack, platforms, what you can deliver |
| `team.md` | Team size, key roles, expertise areas |
| `approach.md` | Your delivery methodology, how you work with clients |
| `security.md` | Data security, compliance certifications (ISO 27001, PCI DSS, SOC2, etc.) |
| `pricing-norms.md` | How you typically structure pricing — useful for commercial proposals |

## How to write them

Just plain text with headings. No special format required. Example:

```
# Technical Capabilities

We build payment infrastructure for banks and fintechs.

## Core areas
- Core banking system integrations
- Payment switch and gateway development
- Mobile banking applications

## Certifications
- PCI DSS Level 1 (valid until March 2026)
- ISO 27001 certified
```

The more detail here, the better Claude can write proposals without asking you questions.
